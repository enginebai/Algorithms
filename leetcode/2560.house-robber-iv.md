# [2560. House Robber IV](https://leetcode.com/problems/house-robber-iv/description/)

## Hints

- What if you could check, for a given capability, whether it's possible to rob at least `k` non-adjacent houses?
- Can you use the monotonicity of the problem to apply binary search on the answer?
- How do you efficiently check if a capability is feasible?

## Breakdowns

> 1. How do we check if a given capability is enough to rob at least `k` non-adjacent houses?

Simulate robbing houses: for each house, if its value is ≤ capability and it's not adjacent to the last robbed house, rob it and skip the next. Count how many you can rob.

> 2. How do we find the minimum such capability?

Notice that if a capability is feasible, any larger capability is also feasible (monotonicity). Use binary search on the possible capability values.

## Key Insights

- This is a `binary search on answer` problem. The answer is the minimum "capability" (max value among robbed houses) such that you can rob at least `k` non-adjacent houses.
- **Monotonicity**: If a capability is feasible, any larger capability is also feasible.
- **Lower bound**: The minimum value in `nums` (since you must rob at least one house).
- **Upper bound**: The maximum value in `nums` (in the worst case, you must rob the most expensive house).
- **Feasibility**: For a given capability, greedily try to rob as many non-adjacent houses as possible with value ≤ capability.

## Binary Search on Value

The key idea is to use binary search to find the smallest capability such that you can rob at least `k` non-adjacent houses. For each guess, use a greedy check to see if it's feasible.

```kotlin
fun minCapability(nums: IntArray, k: Int): Int {
    var left = nums.min()
    var right = nums.max()
    while (left <= right) {
        val middle = left + (right - left) / 2
        var count = 0
        var i = 0
        while (i < nums.size) {
            if (nums[i] <= middle) {
                count++
                i += 2
            } else {
                i++
            }
        }
        if (count >= k) {
            right = middle - 1
        } else {
            left = middle + 1
        }
    }
    return left
}
```
- **Time Complexity**: `O(n log(max(nums)))` where `n` is the number of houses.
- **Space Complexity**: `O(1)`

### Alternative: Two-Start Greedy (Parity)

Some solutions check both starting from index 0 and 1 (parity), but the above greedy is sufficient since the binary search will always find the minimum feasible capability.

## Edge Cases

- All house values are the same.
- `k` is 1 (just need the minimum value).
- `k` is `n/2` or `n` (forces you to take every other or every house).
- Large `nums` values (test for overflow if using `Long`).

## Pitfalls

- Forgetting to check both even and odd starting indices (not needed if greedy is correct, but some implementations do).
- Off-by-one in binary search (`left <= right` vs `left < right`).
- Not updating `i` correctly when robbing a house (should skip the next house).

```kotlin
fun minCapability(nums: IntArray, k: Int): Int {
    var left = nums.min().toLong()
    var right = nums.max().toLong()
    while (left <= right) {
        val middle = left + (right - left) / 2
        val feasible = check(nums, k, middle, 0) || check(nums, k, middle, 1)
        if (feasible) {
            right = middle - 1
        } else {
            left = middle + 1
        }
    }
    return left.toInt()
}

private fun check(nums: IntArray, k: Int, capability: Long, start: Int): Boolean {
    val n = nums.size
    var i = start
    var houses = 0
    while (i < n) {
        if (nums[i].toLong() <= capability) {
            i += 2
            houses++
        } else {
            i++
        }
    }
    return k <= houses
}
```
