# [926. Flip String to Monotone Increasing](https://leetcode.com/problems/flip-string-to-monotone-increasing/)

## Enumeration
A monotone increasing string like `000111`, `0000`, or `1111` always has a "pivot", so that everything before it is `0` and everything after it is `1`. The string can be split into two halves (the pivot itself is the **left** part:
```js
 0, 0, ...   i | 1, 1, 1, ...
[only 0's to i] [only 1's after i]
```

For every pivot, we have to flip everything to `0` before it, and everything to `1` after it, so the number of flips is `(Number of 1s on the left)` + `(Number of 0s on the right)`

```js
0, 1, 0, 1, 0, 0, 1, 1, 0
         i|
   0     0  1  1        1   // Flip
          |
[# of 1's]|[# of 0's]       // The number of flips
          |

// Example:
    0, 1, 1, 0, 0, 0   flips
 i  1        1  1  1 = 0 + 4
    i        1  1  1 = 0 + 3
       i     1  1  1 = 0 + 3
       0  i  1  1  1 = 1 + 3
       0  0  i  1  1 = 2 + 2
       0  0     i  1 = 2 + 1
       0  0        i = 2 + 0 (min)
```

There are some details to pay attentions:
* We include the current index `i` itself to be left part, so we have to flip if it's `1` and don't flip if it's `0`.
```js
0, 1, 0, 1, 0, 0, 1, ...
         i|
   F     F

1, 1, 0, 0, 1, 0, ...
         i|
F  F
```
* We must initialize `minFlips` to be **the number of 0's after `i`**. It represents that left side is entirely empty, and the right side is the entire string, then we have to flip all 0's to 1's. (This is the case we flip the entire string to 1's)

```js
    0, 1, 1, 0, 0, 0   flips
i|  F        F  F  F       4
```

> Q: For a string, we can flip to be "000...1111..." or "000...." or "111...". How do we handle the flip to "000..." or "111..."?
> 
> A: Before the iteration, we initialize `minFlips` to be the number of 0's after `i`, this is for the case to flip to "111...". When `i == n - 1`, we flip the entire string to 0's, this is the case we flip to "000...".

* How to calculate `(Number of 1s on the left)` + `(Number of 0s on the right)`? We use a prefix sum of 0's and 1's for left side. For right side deduction, it's simply `Total 0s - 0s seen so far`:

```kotlin
fun minFlipsMonoIncr(s: String): Int {
    val n = s.length
    val total0 = s.count { it == '0' }
    val total1 = s.count { it == '1' }
    var left0 = 0
    var left1 = 0
    var minFlips = total0
    for (i in s.indices) {
        if (s[i] == '0') left0++ else left1++
        
        val right0 = total0 - left0
        minFlips = minOf(minFlips, left1 + right0)
    }
    return minFlips
}
```

## Enumeration 2
We itearte the array from left to right, and there are some possible cases:
1. Current is `0`, and haven't seen any `1` before it: It's fine, cost = 0.
2. Current is `1`, and haven't seen any `1` yet: It's fine, we keep track of how many `1's` we've seen, they are potential "liabilities" if we see a `0` later.
3. Current is `0` and we've seen `1` before (The Conflict): It breaks the monotone property. We have two choices:
    - Flip the current `0` to `1`: Cost increments by 1.
    - Flip all previous `1's` to `0's`: Cost is the number of `1's` seen so far.
    - We take the minimum of the two costs.

```kotlin
fun minFlipsMonoIncr(s: String): Int {
    var count1 = 0
    var flipsCount = 0
    for (c in s) {
        if (c == '0') {
            val flipCurrent0 = flipsCount + 1
            val flipAllPrevious1 = count1
            flipsCount = minOf(flipCurrent0, flipAllPrevious1)
        } else {
            count1++
        }
    }
    return flipsCount
}
```
